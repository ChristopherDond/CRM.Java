Projeto Integrador: CRM

Java & SQL Server - Integração Completa

Ver Repositório Completo

Propósito Central

O projeto implementa um Sistema de Gerenciamento de Relacionamento com o Cliente (CRM) de base, totalmente focado na persistência de dados.

Motivação: Demonstrar de forma prática a conexão estável e o tráfego de dados eficiente entre uma aplicação back-end (Java) e um banco de dados relacional (SQL Server).

O Escopo Principal é validar o ciclo completo de CRUD (com foco em Inserção e Leitura) e a aplicação dos conceitos de Programação Orientada a Objetos (POO) e Modelagem de Dados.

Diagrama de Arquitetura

Adotamos uma arquitetura de duas camadas simplificadas, onde a lógica de negócios está majoritariamente centralizada no banco de dados.

  • Camada de Apresentação/Lógica (Java): Aplicação de terminal (`main`), classes de Entidade (POO), e a camada DAO (Data Access Object) para o JDBC.
  • Camada de Dados (SQL Server): Tabelas, Esquemas Relacionais e Stored Procedures que executam o CRUD.

Isso garante segurança (previne SQL Injection) e desempenho ao usar procedimentos pré-compilados.

Core Tecnológico

Java (JDBC) - A Lógica da Aplicação

Implementação da lógica, interface de usuário simples (via `JOptionPane` ou console) e a Camada de Acesso a Dados (DAO). O JDBC é o pilar para a comunicação com o SGBD.

Exemplo de Código (Conexão JDBC):
String URL = "jdbc:sqlserver://[Server];..."
Connection conn = DriverManager.getConnection(URL);
// Uso de Prepared/CallableStatement para segurança
// e execução de Stored Procedures.

SQL Server - Persistência Segura

Utilizado como o SGBD para o armazenamento persistente e relacional de todas as entidades críticas do CRM: Clientes, Contatos, Oportunidades e Produtos.

Trecho SQL (Criação da Tabela de Clientes):
CREATE TABLE Clientes (
    ClienteID INT PRIMARY KEY IDENTITY(1,1),
    Nome VARCHAR(100) NOT NULL,
    Email VARCHAR(100) UNIQUE,
    DataCadastro DATE DEFAULT GETDATE()
);

Stored Procedures - Segurança e Desempenho

O método de comunicação principal. Ao centralizar a lógica de Inserção/Atualização no banco (Stored Procedures), ganhamos em segurança (mitiga SQL Injection) e desempenho (código pré-compilado).

Chamada de Stored Procedure (Java):
String call = "{call SP_INSERIR_CLIENTE(?, ?)}";
try (CallableStatement cs = conn.prepareCall(call)) {
    cs.setString(1, cliente.getNome());
    cs.setString(2, cliente.getEmail());
    cs.executeUpdate();
}

O Que o CRM Faz

  • Gerenciamento de Clientes:

    Criação e listagem de clientes essenciais.

  • Vinculação de Contatos:

    Associação de detalhes de contato a um cliente por ID.

  • Cadastro de Produtos:

    Inserção de novos produtos com campos opcionais.

  • Registro de Oportunidades:

    Inserção de registros de venda/oportunidade com status.

  • Auditoria Rápida:

    Exibição dos últimos 5 registros inseridos (teste de `SELECT TOP`).

  • Tratamento de Exceções:

    Gerenciamento robusto de erros de conexão e SQL (`try-with-resources`).

O Que Aprendemos

  • Padrão JDBC: Domínio na conexão e manipulação de dados com Java.
  • Modelagem Relacional: Fortalecimento na criação de entidades e relacionamentos (1:N, N:M e 1:1).
  • POO Prática: Aplicação de classes, atributos e encapsulamento em um contexto real.
  • Segurança: Uso de `CallableStatement` contra SQL Injection.
  • Stored Procedures: Transferência da lógica de negócios para o SGBD.
  • Tratamento de Erros: Implementação de `try-with-resources` e exceções SQL.
